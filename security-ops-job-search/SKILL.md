### 步骤1.5：岗位真实性验证（必须执行，否则不写入文件）

> **核心原则：3个真实岗位 > 10个虚假岗位。宁缺毋滥。**

对每个搜索到的岗位，逐一执行：

3. **存活检查**：用 WebFetch 尝试打开链接，确认页面存在且内容匹配
4. **时效检查**：岗位发布日期是否在最近3个月内？

**三不写原则：**
- ❌ 链接无法验证 → 不写入文件
- ❌ 来源为黑名单平台 → 不写入文件
- ❌ 学历硬性要求本科/硕士且无例外 → 不写入（降级为参考除外）

**验证结果必须记录在岗位详情中：**

| 级别 | 定义 | 可写入文件？ | 写入位置 |

#### B. 验证流程（对每个搜到的岗位逐一执行）

1. **检查链接来源**：搜索结果中的URL属于哪个平台等级？

   - P3-P4 → 进入第2步验证

2. **WebFetch验证**：根据链接等级执行不同程度的验证：

### 步骤2.5：公司风险验证（必须执行）

> **核心原则：匹配度再高的岗位，如果公司本身有风险，也必须降级或排除。每分析一个职位，必须同步进行公司背景核查。**

#### A. 公司工商信息核实

对每个分析到的公司，通过以下方式核查：

1. **天眼查**（tianyancha.com）/ **企查查**（qcc.com）/ **国家企业信用信息公示系统**（gsxt.gov.cn）
   - 使用 WebSearch 搜索 `"{公司全名} 天眼查"` 或 `"{公司全名} 企查查"`
2. **重点检查项：**
   | 检查项 | 正常 | 风险 |
   |--------|------|------|
   | 注册资本 vs 实缴资本 | 实缴≥注册资本50% | 实缴为0或极少 |
   | 参保人数 | 与公司规模匹配 | 参保人数为0-5（可能为空壳或全员外包） |
   | 经营状态 | 存续/在业 | 列入经营异常、严重违法 |
   | 成立时间 | ≥2年 | 不足半年（可能是新壳） |
   | 劳动仲裁 | 无或极少 | 多条劳动仲裁记录（拖欠工资信号） |
   | 行政处罚 | 无 | 多条行政处罚 |

#### B. 外包公司识别

1. **直接识别：** 看公司名称是否在上文列出的外包公司名单中
2. **JD关键词识别：** 职位描述中出现以下词汇需高度警惕：
   - ❌ 高风险："驻场开发""外派""人力外包""劳务派遣""代招""外包岗位"
   - ⚠️ 需确认："驻场运维""客户现场""长期驻场""驻场安全"
3. **区分"外包派遣"和"安全驻场"（重点！）：**
   - 安全驻场（✅ 可接受）：签约主体是安全厂商（奇安信/深信服/绿盟等），缴纳五险一金，本质是做安全运维/应急响应/安全服务。安全运维岗的驻场是行业常态，不是外包
   - 外包派遣（❌ 需排除）：签约主体是外包公司而非安全厂商，派遣到甲方做杂活，无归属感无技术成长

#### C. 骗子公司/虚假招聘识别

出现以下任一特征，标记为 🔴高风险：
- **培训贷陷阱：** JD或面试中提到"网络安全培训""岗前培训""培训后包就业""先培训后付费"
- **虚假高薪：** 薪资远超市场水平（应届生安全运维正常范围5-12K，承诺15K+且无经验要求需警惕）
- **收钱招聘：** 任何形式的收费（押金、培训费、服装费、体检费、工牌费）
- **办公地址异常：** 在居民小区、虚拟地址、无固定办公场所
- **公司刚注册：** 工商注册不到半年，且无实际安全业务
- **面试流程异常：** 一轮面试即录用、面试官不像技术人员、面试地点不在公司

#### D. 薪资保障评估

1. **员工口碑查询：**
   - 使用 WebSearch 搜索 `"{公司名} 拖欠工资"` `"{公司名} 工资 怎么样"` `"{公司名} 脉脉"`
   - 看准网（kanzhun.com）/ 脉脉搜索员工评价（薪资、加班、值班、氛围）
2. **融资/经营状况：**
   - 安全厂商通常有稳定客户收入（政府/金融/运营商），稳定性较好
   - 小型安全服务公司需查成立年限和客户案例
3. **面试时委婉确认：**
   - 可问："公司社保公积金是按实际工资缴纳吗？"
   - 可问："薪资是固定发放还是有绩效浮动？每个月几号发工资？"
   - 可问："值班/夜班有额外补贴吗？标准是什么？"
   - 避坑：如果HR对薪资结构含糊或说"转正后再说"，立刻终止

- **学历劣势应对：** 安全运维岗位最看重实战能力而非学历。面试时主动展示：处理过的安全事件数量、运维的设备规模、写过的自动化脚本效果

## 🛡️ 求职安全守则（每次搜索必须遵守）

### 投递前必查清单
每投递一个岗位前，确认以下事项：
- [ ] 已用天眼查/企查查核实公司工商信息（注册时间、实缴、参保人数、风险记录）
- [ ] 已搜索"{公司名} 拖欠工资"确认无负面记录
- [ ] 已看脉脉/看准网员工评价（安全运维岗尤其关注是否频繁加班/值班无补偿）
- [ ] 确认签约主体不是外包/派遣公司（安全厂商驻场✅，人力外包❌）
- [ ] 确认JD中无"培训后上岗""先培训再入职""安全培训包就业"等字眼
- [ ] 确认面试地点与工商注册地址一致
- [ ] 确认无任何形式的收费要求

### 面试安全提示
- **面试地点：** 只去公司办公场所面试，拒绝去酒店/咖啡厅/居民楼面试
- **收钱就跑：** 面试过程中任何形式的收费（培训费、押金、工装费等）都是骗局，立即终止
- **劳动合同：** 入职当天必须签劳动合同，看清楚签约主体公司名称、试用期时长、薪资结构与JD承诺是否一致
- **五险一金：** 问清楚缴纳基数和比例（按实际工资缴纳还是最低工资），入职第一个月就应开始缴纳
- **警惕模糊岗位名：** "网络安全工程师（管培生）""储备安全干部"等名称模糊的，可能不是真正的安全运维岗
- **薪资结构：** 问清楚底薪+绩效+奖金的构成，拒绝"转正后再说""看能力定薪"等模糊说法
- **值班待遇：** 安全运维岗常涉及7x24值班，面试时问清楚值班频率、是否有夜班补贴和调休

### 绝不投递的红线
出现以下任何一条，直接放弃该岗位：
1. 公司是外包/劳务派遣公司（非安全厂商正式驻场编制）
2. JD或面试中提到"岗前培训""先培训"
3. 任何形式的收费
4. 工商状态异常（经营异常、严重违法、大量劳动仲裁）
5. 公司成立不到半年且查不到任何产品/业务信息
6. 有拖欠工资的明确记录
7. 岗位投递链接无法验证（404/内容不匹配/无法找到原始JD），但岗位本身信息可用——此时降级为参考，不写入"投递链接"
